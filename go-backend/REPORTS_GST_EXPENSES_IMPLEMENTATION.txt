================================================================================
TECHNICAL DETAILS
================================================================================

Data Types & Precision:
- decimal.Decimal for all financial calculations (no floating-point errors)
- Proper null handling with pointer types (*string, *int, *time.Time)
- Time zone aware date/time handling (UTC)

Database Operations:
- GORM ORM for database access
- Efficient preloading to minimize queries
- In-memory aggregation using maps for complex grouping
- MySQL-compatible SQL syntax (DATE_FORMAT, IFNULL)
- Proper filtering with date ranges and status conditions
- Pagination support for large datasets

Key Features:

Reports:
- Revenue analysis and trending
- Customer segmentation
- Payment method distribution
- Aging analysis (0 days, 1-30, 31-60, 61-90, 90+)
- CSV exports with UTF-8 BOM for Excel

GST:
- B2B vs B2CS segregation
- CGST/SGST split (50-50 for intra-state)
- IGST handling (inter-state)
- HSN code grouping
- ITC tracking and carry-forward
- Tally XML export for ERP integration

Expenses:
- Multi-component GST (CGST, SGST, IGST)
- Supply type tracking (INTRA_STATE, INTER_STATE)
- Recurring automation with recurrence patterns
- Monthly budget tracking
- ITC eligibility marking
- Status workflow (PENDING → APPROVED/REJECTED)

Incentives:
- Multi-tier rule support
- Staff-specific or org-wide rules
- Automatic payout calculation
- Leaderboard generation

================================================================================
INTEGRATION NOTES
================================================================================

Architecture:
- Service layer: Business logic, database operations
- Handler layer: HTTP request/response, parameter parsing, error handling
- Middleware: Authentication, authorization, error handling
- Response format: Consistent JSON responses via util.SendSuccess/SendError

Dependencies:
- GORM ORM (database)
- decimal package (decimal.Decimal)
- Standard library (encoding/json, net/http, time)
- Existing models (Order, Invoice, PurchaseOrder, Expense, etc.)

Error Handling:
- Returns error as second return value
- Handlers convert errors to HTTP responses
- Uses existing handleError() helper

Middleware Integration:
- Follows existing middleware chain pattern
- Compatible with Authenticate() and RequireRole()
- Proper HTTP status codes
